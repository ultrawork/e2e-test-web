# E2E-сценарии: Web v23 — /notes и авторизация

**Версия:** v23
**APP_PORT:** 3000 (Next.js)
**API_PORT:** 4000 (backend)
**Переменная окружения:** `NEXT_PUBLIC_API_URL=http://localhost:4000`
**Авторизация:** токен хранится в `localStorage` под ключом `token`

---

## SC-v23-01: Без токена — показ требования авторизации

**Предусловие:** Токен не установлен в localStorage.
**Шаги:**
1. Открыть `/notes`.
2. Проверить наличие текста «Необходима авторизация».
3. Проверить наличие ссылки «Войти».
4. Убедиться, что поле ввода заметки (`placeholder="Enter a note"`) не отображается.

**Ожидаемый результат:** Страница показывает требование авторизации, интерфейс заметок скрыт.

---

## SC-v23-02: С токеном — интерфейс заметок доступен

**Предусловие:** В localStorage установлен валидный токен (dev-token). API /api/notes возвращает 200.
**Шаги:**
1. Установить токен в localStorage: `localStorage.setItem('token', '<dev-token>')`.
2. Открыть `/notes`.
3. Проверить заголовок «Notes».
4. Проверить поле ввода (placeholder «Enter a note»).
5. Проверить кнопку «Add».
6. Проверить счётчик «Всего заметок: 0».

**Ожидаемый результат:** Все элементы интерфейса заметок отображаются корректно.

---

## SC-v23-03: Создание заметки увеличивает счётчик

**Предусловие:** Авторизован (токен в localStorage), страница `/notes` открыта, счётчик = 0.
**Шаги:**
1. Ввести текст «E2E тест v23» в поле ввода.
2. Нажать кнопку «Add».
3. Проверить: заметка отображается, счётчик = 1.
4. Ввести текст «Вторая заметка v23».
5. Нажать кнопку «Add».
6. Проверить: счётчик = 2.

**Ожидаемый результат:** Каждое добавление увеличивает счётчик на 1.

---

## SC-v23-04: Удаление заметки уменьшает счётчик

**Предусловие:** Авторизован (токен в localStorage), страница `/notes` открыта.
**Шаги:**
1. Добавить заметку «Заметка для удаления v23».
2. Добавить заметку «Остаётся v23». Счётчик = 2.
3. Нажать кнопку удаления для «Заметка для удаления v23».
4. Проверить: заметка исчезла, счётчик = 1.
5. Нажать кнопку удаления для «Остаётся v23».
6. Проверить: счётчик = 0.

**Ожидаемый результат:** Каждое удаление уменьшает счётчик на 1.

---

## SC-v23-05: api.ts добавляет заголовок Authorization: Bearer

**Предусловие:** Токен установлен в localStorage.
**Шаги:**
1. Перехватить запрос к `/api/notes` (Playwright route interception).
2. Открыть `/notes`.
3. Проверить заголовок `Authorization` в перехваченном запросе.

**Ожидаемый результат:** Заголовок содержит `Bearer <token>`, где `<token>` — значение из localStorage.

---

## SC-v23-06: api.ts корректно обрабатывает 401

**Предусловие:** Токен установлен в localStorage. API /api/notes возвращает 401.
**Шаги:**
1. Перехватить запрос к `/api/notes` и вернуть 401.
2. Открыть `/notes`.
3. Проверить отображение «Необходима авторизация».
4. Проверить, что токен удалён из localStorage.

**Ожидаемый результат:** Токен удаляется, показывается экран авторизации.
