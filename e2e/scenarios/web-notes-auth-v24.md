# E2E-сценарии: Web Notes Auth v24

Верификация страницы `/notes` и модуля `api.ts` — авторизация, Authorization header, обработка 401.

**Платформа:** Web (Next.js)
**Инструмент:** Playwright
**API-моки:** `page.route()` (backend не требуется)

---

## SC-001: Без токена — гейт авторизации

| Поле | Значение |
|------|----------|
| **Тип** | Authorization gate |
| **Приоритет** | High |
| **Режим** | automated (Playwright) |

**Предусловия:**
- localStorage не содержит `token`

**Шаги:**
1. Очистить localStorage (`localStorage.clear()`)
2. Перейти на `/notes`

**Ожидаемый результат:**
- Отображается текст "Необходима авторизация"
- Отображается ссылка "Войти" с `href="/login"`
- Поле ввода `#new-note` НЕ отображается

---

## SC-002: С токеном — список заметок из API

| Поле | Значение |
|------|----------|
| **Тип** | Notes list (GET) |
| **Приоритет** | High |
| **Режим** | automated (Playwright) |

**Предусловия:**
- `localStorage.token = 'test-token-v24'`
- Мок GET `/api/notes` возвращает `[{id: '1', text: 'Заметка из API', createdAt: '...'}]`

**Шаги:**
1. Установить токен через `addInitScript`
2. Настроить мок `page.route()` для GET `/api/notes`
3. Перейти на `/notes`

**Ожидаемый результат:**
- Отображается текст "Заметка из API"
- Отображается "Всего заметок: 1"

---

## SC-003: Создание заметки — счётчик растёт + Authorization в POST

| Поле | Значение |
|------|----------|
| **Тип** | Create note (POST) + Auth header |
| **Приоритет** | High |
| **Режим** | automated (Playwright) |

**Предусловия:**
- `localStorage.token = 'test-token-v24'`
- Мок GET `/api/notes` возвращает `[]`
- Мок POST `/api/notes` перехватывает заголовки и возвращает `201` с новой заметкой

**Шаги:**
1. Установить токен и моки
2. Перейти на `/notes`, убедиться "Всего заметок: 0"
3. Ввести текст в `#new-note`, нажать "Add"

**Ожидаемый результат:**
- Новая заметка отображается в списке
- Счётчик обновляется до "Всего заметок: 1"
- Перехваченный POST-запрос содержит заголовок `Authorization: Bearer test-token-v24`

## SC-006: 401 от API — очистка токена + редирект на /login

| Поле | Значение |
|------|----------|
| **Тип** | Error handling (401 Unauthorized) |
| **Приоритет** | Critical |
| **Режим** | automated (Playwright) |

**Предусловия:**
- `localStorage.token = 'test-token-v24'`
- Мок GET `/api/notes` возвращает `401 {"error": "Unauthorized"}`

**Шаги:**
1. Установить токен через `addInitScript`
2. Настроить мок, возвращающий 401
3. Перейти на `/notes`

**Ожидаемый результат:**
- Происходит редирект на `/login`
- `localStorage.getItem('token')` возвращает `null`
